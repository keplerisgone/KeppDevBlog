
# 리눅스 개발이 필요한 이유

- 메모리 / 비메모리 반도체
	- 메모리 반도체 (용량이 중요)
    - RAM, ROM 등
    - 소품종 대량 생산, 구조가 단순
    - 우리나라 수출 대부분
	- 비메모리 반도체 = 시스팀 반도체, SW 필요
    - CPU, GPU, Comm 등
    - 시스템 반도체
    - 다품종 소량 생산 (CPU를 생각해보면..), 구조가 복잡
    - 많은 부가가치 창출 가능
    - 반도체 위에서 구동될 SW 제공 필요

- 팹리스 반도체 회사
	- 설계와 판매 전문, 제조는 TSMC에 맡김

=> 결국 반도체의 설계와 반도체 위에서 구동되는 SW가 중요

## SOC (System On Chip)

- 비메모리 반도체 개발의 흐름
- 각각의 장치를 하나의 칩으로 만드는 방식
  - CPU, GPU와 같이 핵심적인 부분 뿐만 아니라 WIFI/USB 등과 같은 부가 기능도 한 칩에 포함시키기도 함
  - ROM, RAM은 따로 하기도 함
- 제품 소형화 가능
	- 복잡한 시스템 구현 X
	- 버스 구현 정교화 가능
- SoC 위의 장치는 칩제조사가 제공하므로 각각의 디바이스 드라이버를 칩 제조사가 제공해주어야 
- 예시 ) 니가 쓰는 M1

### SoC 개발 흐름

- 칩 + SW가 동시에 개발됨
- 칩이 없는 시점에도 에뮬을 이용해 SW를 개발한다.

# 왜 리눅스를 써야 하는가

- 양질의 코드를 활용가능 (오픈 소스라)
- 구매 비용이 없다 (no liscense)
- 다양한 개발도구 지원 (gcc 같은 것)
